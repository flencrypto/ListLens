This is the mobile route because it turns RecordLens into a camera-first listing machine: snap the vinyl, resolve the Discogs release, map the fields, then generate an eBay-ready draft.

Best mobile setup

Use Expo / React Native as the front end. expo-image-picker supports camera and photo-library capture, so it fits perfectly for record sleeves, labels, barcodes and matrix shots. 

The key rule

Do not put Discogs secrets in the mobile app.

Use the app as the capture layer, then send requests to a Netlify Function:

Mobile app
  ↓ photo / barcode / title / matrix
Netlify Function
  ↓ signed Discogs request
Discogs API
  ↓ release metadata
RecordLens field mapper
  ↓
Listing draft

Expo warns that EXPO_PUBLIC_ variables are visible in the compiled app, so Discogs keys, OAuth secrets and signing logic must stay server-side.

Tiny mobile client shape

// app/services/recordlens.ts

export async function analyseRecord(formData: FormData) {
  const res = await fetch(`${process.env.EXPO_PUBLIC_API_URL}/analyse-record`, {
    method: "POST",
    body: formData,
  });

  if (!res.ok) {
    throw new Error("RecordLens analysis failed");
  }

  return res.json();
}

Only store short-lived user/session tokens on-device. For sensitive local values, Expo SecureStore encrypts key-value pairs locally on the device.
